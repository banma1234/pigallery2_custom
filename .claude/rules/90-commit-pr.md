# 커밋 및 PR 규칙

## 커밋 메시지 형식

```
[타입] 제목

본문 (선택사항)

Footer (선택사항)
```

## 커밋 타입

| 타입 | 용도 |
|------|------|
| `[Add]` | 새로운 파일, 기능, 테스트 추가 |
| `[Feature]` | 새로운 기능 구현 |
| `[Fix]` | 버그 수정 |
| `[Refactor]` | 코드 리팩토링 (기능 변경 없음) |
| `[Style]` | 코드 포맷팅, 세미콜론 등 (로직 변경 없음) |
| `[Docs]` | 문서 수정 |
| `[Test]` | 테스트 코드 추가/수정 |
| `[Chore]` | 빌드 업무, 패키지 매니저 설정 등 |
| `[Remove]` | 파일, 코드 삭제 |
| `[Rename]` | 파일명, 변수명 변경 |
| `[Move]` | 파일 이동 |

## 작성 규칙

- 제목은 50자 이내, 명령문, 끝에 마침표 금지.
- 본문은 72자마다 줄바꿈.
- 본문은 "무엇을", "왜" 변경했는지 설명.

## PR 규칙

- 작고 집중된 단위로 구성.
- 설명 필수 포함: 무엇을 / 왜 / 어떻게 / 검증 방법.
- PR 제목은 커밋 메시지 형식 준수.
